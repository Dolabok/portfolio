Fournir une interface de programmation


Définition

    Fournir une interface de programmation, ou API (pour Application Programming Interface), est une compétence qui implique la création d'outils et de protocoles permettant à divers logiciels et services de communiquer entre eux.
Les API sont utilisées pour accéder aux fonctionnalités d'un système et partager des données entre différents programmes et services. Les développeurs utilisent les API pour faciliter l'intégration et la collaboration entre les applications.


Contexte professionnel

    Dans un contexte professionnel, fournir une API consiste à créer des interfaces pour faciliter l'accès aux données et aux fonctionnalités d'une application ou d'un service. Les API sont essentielles pour assurer l'interopérabilité entre les systèmes et permettre une communication fluide entre différentes applications.
Elles sont également utilisées pour faciliter le développement et l'extension de projets, en simplifiant l'accès aux ressources et en définissant des normes pour la communication.


Actualité

    La demande pour les développeurs capables de créer des API solides et efficaces est en constante augmentation, en particulier avec la montée en puissance des microservices et des architectures orientées services.
Les entreprises recherchent des professionnels capables de concevoir des API modulaires, évolutives et sécurisées pour soutenir leurs opérations et leur croissance.


Réalisation


API DigiTrack d'Altaroad

    Lors de mon travail chez Altaroad, j'ai ajouté de nombreux servir à l'API de DigiTrack avant de participer à la migration que l'horrible Mybatis à JPA, le projet est lié aux chantiers du Grand Paris et aux Jeux Olympiques de 2024.
Cette API doit permettre de manipuler des données clients sensibles et être capable de gérer un volume important d'informations. Pour répondre à ces besoins, nous avons choisi d'utiliser Java Spring Boot JPA pour développer une API solide, sécurisée et évolutivement.
Le succès de cette réalisation a permis à l'entreprise de gérer efficacement les données des chantiers et de respecter les normes de sécurité tout en ajoutant facilement de nouveaux client et toutes leurs spécificitées y compris pour les clients internationaux.


L'utilisation de TypeScript et des fonctions sans serveur

    Dans le cadre de projets nécessitant des API plus légères et rapides à développer, j'ai opté pour TypeScript, qui permet une mise en œuvre rapide et efficace.
Pour des besoins ponctuels précis, j'ai également utilisé des fonctions sans serveur stockées dans le cloud, offrant une flexibilité et une évolutivité accrues.
